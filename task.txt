Постановка задачи.
Разработать объектную программу для хранения и обработки данных о книгах, выставляемых на продажу в магазине. 
Книжный магазин - набор тематических отделов с уникальными названиями. 
Каждый отдел использует свой каталог книг, в котором каждая книга имеет уникальное название, автора и параметр «Число экземпляров».
Разработка включает в себя:
-определение необходимых объектов и способов их взаимодействия
-формальное описание объектов в виде классов
-программную реализацию всех необходимых методов, включая:
    -закуп книги
    -продажу книги
    -подсчет суммарного числа наименований книг 
    -суммарного числа их экземпляров 
    -как по каждому отделу, 
    -так и по магазину в целом
    -всестороннее тестирование методов с помощью консольного (при разработке) и оконного (в окончательном варианте) приложения.

Для объединения отделов используется структура данных в виде адресного замкнутого (кольцевого) неупорядоченного однонаправленного списка с заголовком. 

Для объединения книг в каталог отдела используется кольцевая очередь на основе обычного массива.
